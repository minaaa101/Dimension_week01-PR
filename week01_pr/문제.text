1. Git과 GitHub의 차이점은?
Git은 파일의 변경 이력을 기록하고 관리하는 버전 관리 도구
GitHub는 Git을 바탕으로 한 협업 관리 서비스

2.브랜치를 사용하는 이유는?
브랜치 : 기존 코드에 영향을 주지 않고 새로운 작업을 할 수 있는 독립된 개발 라인
Main 브랜치에서 새로운 브랜치를 생성하여 안전하게 코드 작업 후 병함

3.git push는 어디로 보내는걸까?
내가 수정한 코드를 github에 업로드

4.clone과 frok의 차이점은?
clone이 내 컴퓨터로 작업 폴더를 가져오는 과정이라면 
fork는 상대 github repository를 내 repository로 가져오는 과정

5."git switch -c 브랜치 이름"이 어떤 기능을 하는 코드일까?
branch 생성 + 이동
