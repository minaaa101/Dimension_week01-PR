1. Git과 GitHub의 차이점은?
- Git: 파일의 변경 이력을 기록, 관리하는 버전 관리 도구
Github: Git을 기반으로 하는 원격 저장소 호스팅 서비스

2.브랜치를 사용하는 이유는?
- 기존 코드에 영향을 주지 않고 새로운 작업을 하기 위해 사용

3.git push는 어디로 보내는걸까?
- 로컬 저장소 -> Github

4.clone과 fork의 차이점은?
- fork: 타인의 저장소를 내 Github 계정에 복사
clone: 원격 저장소를 내 컴퓨터에 복사

5."git switch -c 브랜치 이름"이 어떤 기능을 하는 코드일까?
- 새 브랜치 생성 후 교체
