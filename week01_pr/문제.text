1. Git과 GitHub의 차이점은?
    Git은 파일 변경 이력을 기록하고 관리하는 '버전 관리 도구'이고
    GitHub는 Git으로 관리하는 프로젝트를 공유하고 렵업할 수 있게 하는 '플렛폼'이다.
2.브랜치를 사용하는 이유는?
    기존 코드에 영향을 주지 않고 새로운 작업을 하기 위해서.
3.git push는 어디로 보내는걸까?
    GitHub에 업로드 된다.
4.clone과 frok의 차이점은?
    clone은 원격 저장소를 내 컴퓨터로 복사하는 것이고
    fork는 다른 사람의 저장소를 내 GitHub 계정으로 복사하는 것이다.
5."git switch -c 브랜치 이름"이 어떤 기능을 하는 코드일까?
    브랜치 이름이 없다면 브랜치 이름을 만들고 거기로 이동하는 코드.
